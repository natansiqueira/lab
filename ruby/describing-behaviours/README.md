# Describing behaviors
Behavior is a unit of code usually encapsulated in a function or method that describes an action.

* Global Methods
    - Available in all application scopes. It's dangerous to use this kind of behavior.

* Class Methods
    - Available in a Class definition not in their instances

* Instance Methods
    - Available in a Class Instance (after using .new)

* Module Methods
    - Available as static methods

* Functions
    - Using Proc
